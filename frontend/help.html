Markers

Allele data is inputed in the form of values separated by commas. The order in which the alleles are entered does not matter, 14,19.3 and 19.3,14 being for instance equivalent. Concerning homozygous loci, entering the alleles one or more times does not matter since the alleles are represented as a set of unique values. For instance, entering 12,12 or 12 is equivalent. In order to be able to support cell lines with microsatellite instability, there is no limit in the number of alleles that can be entered for a given marker. Note that allele data is whitespace insensitive.

Results

API

The STR Similarity Search Tool API allows to perform a STR profile search directly and retrieve the matching cell lines in the JSON format

Base address

http://129.194.71.205:8080/str-sst/api/query?

Parameters

The parameters are added after the “?” character in the base address and separated by “&” characters

marker "name"
alleles separated by commas

scoring
1 = Tanabe Algorithm
2 = Masters Algorithm (vs. query)
3 = Masters Algorithm (vs. reference)
(default = 1)

mode
1 = Non-empty markers
The score is computed on the markers that have allele data for both the query and the reference
2 = Query markers
The score is computed on the markers for which the user inputed allele data even if the reference is missing allele data (default = 1)

includeAmelogenin
boolean defining if Amelogenin is taken into account for the score computation
(default = false)

filter
integer defining the minimum score
(default = 60)

size
integer defining the maximum number of results
(default = 200)

format
string [json|csv] defining the format of the api response
(default = json)

Example

http://129.194.71.205:8080/str-sst/api/query?Amelogenin=X&CSF1PO=11,12&D13S317=11,12&D16S539=11,12&D18S51=13&D19S433=14&D21S11=29,30&D2S1338=19,23&D3S1358=15,17&D5S818=11,12&D7S820=10&D8S1179=10&FGA=20,22&Penta_D=11,13&Penta_E=14,16&TH01=6,9&TPOX=8,9&vWA=17,19&scoring=1&mode=1&includeAmelogenin=false&size=200&filter=70&format=json
